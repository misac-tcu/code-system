The Code System for FHIR Structured Report

With amount of code systems, libraries and value sets that are already available. However, there is still no international standard code used in medical finding observation, both in mammography, chest x-ray and others. In addition, there are difficulties in mapping code values used in the finding form to LOINC or SNOMED code system. Therefore HL7 establishes an independent value set in breast radiology, but it will lead to different body locations and organs have each own value set, this causes difficulty managing the current system code.

Therefore, a new standardize of code value based on medical anatomy and pathology is created in this specification to facilitate the easy reading and effective definition of code. Moreover, a new standardize of code value obtain the code that can be used for all body sites, it will simplify the use of independent value sets at each body site.

The advantages of the new standardize of code value:
♢ Mnemonic code will be more convenience for communication and easy to understand.
♢ May be used for training AI systems in professional speech recognition in the future
♢ New code system use clear name to define the code value

Example: In current situation there are three different code systems used to define the location code value of liver, breast and colon.
♢ Breast: http://hl7.org/fhir/us/breastcancer/2018May/CodeSystem-oncology-BreastSiteCS.html
♢ Liver:https://www.radelement.org/home/elements/element/RDE1053
♢ Colon: https://www.radelement.org/home/elements/element/RDE79
But for the present it will develop an independent location code system to collect all location value codes used in all body organ (both liver, innestine, and breast radiography, etc), bellow is the Location Code System web link: http://127.0.0.1:5500/newCodeSystem/LocationCS.html
